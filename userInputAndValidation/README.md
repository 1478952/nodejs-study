## 검증 이해

### 검증 사용 이유

왜? 웹앱에는 수많음 폼이 존재함 (로그인 폼, 상품추가 폼) 해당 폼이 post요청으로 백엔드에 전송되면 DB에 데이터를 저장하게 됨.

### 입력 검증 방법

1. js활용 브라우저단 검증방법

- 요청을 보내기전 키 이벤트를 감지해서 사용자 폼 입력 도중 검증, 런타임에서 에러를 발생가능
- 사용자경험이 증가함
- 사용자가 js를 끌수 있기때문에 선택사항임.
- 데이터가 서버에오는것을 완전하게 막아주지 못함

2. 서버측 검증방법

- 사용자가 코드를 보거나 수정 할 수 없음.
- 필수사항임.
